HTML notes

  • Hypertext Markup Language (HTML) is a basic formatting language widely used on the web
  • HTML gives a webpage structure that can be rendered by browsers
  • Elements are the individual parts of an HTML page's structure
  • Each element is encapsulated with unique tags that signals to the browser which element is envoked
  • Elements have attributes, which are values that modify the element's behavior
  • Elements are also broken down into meta-related, content divison, and semantic categories
  • Strong HTML developers attempt to optimize for search engines (SEO), considers aesthetics, readability, and accessibiltiy

CSS notes

  • Cascading Style Sheets (CSS) are companion attributes for HTML elements
  • CSS offers three different scopes to modify an element
    • Inline CSS, which applies to a specific HTML tag using the "style" attribute
    • Internal CSS style sheet, embedded into the head element and modifies any element of that class
    • External CSS style sheet, which is imported with the link tag and contains changes that applies to the whole page
  • A property with a value is called a "declaration"
    • A CSS property has a value which is set, such as {color: blue;}
  • CSS Box Model: Margin{Border{Padding{Content}}}

Git notes

  • Useful cmds:
    • git [-h | --help] [-a | --all] {cmd} : prints common commands, ALL git cmds, or man page for {cmd}
    • git status : reports branch & commit status
    • git clone {URL} : clone a repository to the working directory
    • git pull origin {branch} : syncs local {branch} clone with remote repository of {branch}, {branch} = main is "rebasing" code
    • git checkout [-b | --build] {branch} : switch to {branch}, -b flag to build new branch with name {branch}
    • git add [-A | --all] : stages changed files for later commit
    • git push [origin] {branch} : apply changes to {branch}
    • git commit [-m | --message] "{imperative commit message}"
    • git revert {hash} : (careful!) version hash to revert branch to
    • git diff --check : lists possible whitespace errors
    • git config [-g | --global] user.email "you@example.com" : configures local git to reference your email when you commit
    • git config [-g | --global] user.name "Your Name" : set name associated with commits from local changes

JavaScript notes

  • JavaScript (JS) is an object-oriented scripting & programming language which is interpretted at run-time
  • JS does not require compilation like other programming languages, like C++
  • Shares a few features in common with Python, such as immutable strings & auto detected variable types
  • It's best practice to use let to define variables, as the var keyword defines a global scope variable
  • JS supports string interpolation
